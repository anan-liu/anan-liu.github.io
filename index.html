strObject

发表于 2023-09-29 更新于 2023-09-30 分类于 redis

字符串对象

Redis对象有三个主要属性：type、encoding、ptr。字符串对象type的类型就是字符串：REDIS_STRING

字符串的编码有：1、int；2、raw；3、embstr

字符串对象根据以下的规则选择编码存储数据

类型 encoding ptr
整数&可以使用long来保存 int ptr->long*
字符串&长度>32bytes raw ptr->SDS*
字符串&长度<32bytes embstr ptr->SDS*

embstr

其中，embstr的特点是：对象和ptr指向的数据结构存放于一块连续的内存空间。也即是，声明一个编码位embstr的对象时，仅需要系统分配一次内存空间，释放内存时也仅需要释放一次，提高了效率。

还需注意的是：embstr没有对应的写接口，只有读接口，所以当修改了embstr编码的对象时，Redis会将embstr编码转为raw编码，再进行修改，此时对象也转变为raw编码了。

int

由上表可知，对象使用int编码时ptr指向的内存必须保存着整数型的数据，因此:

  • 一旦向这个整数型的数据添加进字符串内容时，编码转为raw
  • 这个数据是浮点型的，则编码为raw，但是做计算时仍然可以实现浮点运算功能

raw

除去上述的例外，其余的情况编码均为raw。

初始化配置

刚装上git的机器，需要做一个初始化。git的配置信息存储在三个文件中，三个文件的作用范围分别为：对所有用户生效，对当前用户生效，对当前仓库生效

  • 对所有用户生效：/etc/gitconfig,执行git config 带上 --system选项，命令的操作对象就是这个文件
  • 对当前用户生效：~/.gitconfig/,执行git config 带上 --global选项，命令的操作对象就是这个文件
  • 对当前仓库生效：.git/config,执行git config 带上 --local选项，命令的操作对象就是这个文件

仓库

仓库（Repository）是存储和管理项目代码的地方。它可以位于本地计算机上，也可以位于远程服务器上。

仓库状态

  • 已提交（committed）:已修改表示修改了文件，但还没保存到数据库中。

  • 已修改（modified） :已暂存表示对一个已修改文件的当前版本做了标记，使之包含在下次提交的快照中。

  • 已暂存（staged）:已提交表示数据已经安全地保存在本地数据库中。
