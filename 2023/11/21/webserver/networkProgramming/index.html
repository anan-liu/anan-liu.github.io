IP地址

套接字地址结构

套接字 = IP＋端口

  • sockaddr_in 是用于表示IPv4地址的结构体
  • sockaddr 是一个通用的套接字地址结构体

在使用套接字编程时，通常会将 sockaddr_in 转换为 struct sockaddr，以便在函数参数中传递。这是因为很多套接字相关的系统调用（如 bind、connect、accept 等）都接受通用的 struct sockaddr 类型的地址参数。

地址转换

getaddrinfo()

  • node：主机名或 IP 地址字符串，可以为 NULL。
  • service：服务名或端口号字符串，可以为 NULL。
  • hints：一个可选的指向 struct addrinfo 结构体的指针，用于提供关于期望返回的地址信息类型的提示。如果不需要特殊的提示，可以将其设置为 NULL。
  • res：一个指向 struct addrinfo 结构体链表的指针，返回获取到的地址信息。

在使用 getaddrinfo() 函数获取地址信息之后，需要遍历返回的地址信息链表并调用 socket()，这是因为一个主机名（或服务名）可能对应多个地址，而不同的地址可能代表不同的网络接口或协议。

struct addrinfo

以上的结构体就是地址信息链表：struct addrinfo 结构体中的字段描述了一个网络地址的各种属性。

  • ai_flags 控制返回信息的标志，
  • ai_family 表示地址族，
  • ai_socktype 表示套接字类型，
  • ai_protocol 表示协议，
  • ai_addrlen 表示地址长度，
  • ai_addr 是一个指向 sockaddr 结构体的指针，包含实际的地址信息。
  • ai_canonname 存储主机的规范名，
  • ai_next 指向链表中的下一个 addrinfo 结构体。

inet_ntop()

  • af：地址族，可以是 AF_INET（IPv4）或 AF_INET6（IPv6）。
  • src：指向包含二进制地址的指针，通常是 struct sockaddr 结构体中的 sa_data 字段。
  • dst：指向存储转换后的字符串的缓冲区。
  • size：缓冲区的大小。

套接字选项

setsockopt()

是一个系统调用，用于设置套接字选项。套接字选项是影响套接字行为的参数

  • sockfd：套接字描述符，标识要设置选项的套接字。
  • level：指定选项的级别，常见的有 SOL_SOCKET（通用套接字选项）、IPPROTO_TCP（TCP 选项）和 IPPROTO_IP（IP 选项）等。
  • optname：指定要设置的选项的名称，具体取决于 level 的值。
  • optval：指向包含新选项值的内存块的指针。
  • optlen：optval 指向的内存块的大小。

当 level 设置为 SOL_SOCKET 时，optname 和 optval 参数的含义会根据具体的选项而变化。

socket() && connect()

服务端

socket() && bind() && listen() && accept()

getpeername

getpeername() 是一个用于获取已连接套接字对端地址信息的系统调用。它通常用于在服务器端获知与其连接的客户端的地址信息。

  1. sockfd：
    • sockfd 是已连接套接字的文件描述符，它是通过 socket() 和 accept() 等函数创建的。
  2. addr：
    • addr 是一个指向 struct sockaddr 结构体的指针，用于存储对端（远程端）的地址信息。
  3. addrlen：
    • addrlen 是一个指向 socklen_t 类型的指针，用于指定 addr 缓冲区的大小。在调用 getpeername() 之前，addrlen 应该被设置为 sizeof(struct sockaddr)。

getpeername() 在成功时返回 0，失败时返回 -1，并设置全局变量 errno 来指示错误的类型。

消息

struct msghdr

struct msghdr 是用于描述消息的数据结构，通常在网络编程中的一些系统调用中使用，如 sendmsg() 和 recvmsg()。

  • msg_name 和 msg_namelen：
    • msg_name 和 msg_namelen 用于指定目标地址的信息。通常，msg_name 是一个指向 struct sockaddr 结构体的指针，msg_namelen 是这个结构体的长度。
  • msg_iov 和 msg_iovlen：
    • msg_iov 是一个指向 struct iovec 结构体数组的指针，用于指定发送或接收的数据块。msg_iovlen 是 msg_iov 数组的元素个数。struct iovec 包含了指向数据的指针和数据的长度。
  • msg_control 和 msg_controllen：
    • msg_control 和 msg_controllen 用于传递辅助数据，比如控制信息或文件描述符。这个字段通常用于在消息中传递额外的信息。
  • msg_flags：
    • msg_flags 用于传递或接收标志信息，通常用于设置或获取一些与消息相关的状态。

struct cmsghdr

（Control Message Header）是一个用于描述控制消息的结构体头部，通常在一些套接字相关的系统调用中使用，如 sendmsg() 和 recvmsg()。

  • cmsg_len：
    • cmsg_len 表示整个控制消息的总长度，包括 cmsghdr 结构体头部以及后续的实际数据。
  • cmsg_level：
    • cmsg_level 指定协议层，例如 SOL_SOCKET 表示套接字层。
  • cmsg_type：
    • cmsg_type 指定消息的类型，表示控制消息的具体含义。

宏

用于处理和操作控制消息的不同方面。这些宏的命名通常以 CMSG_ 为前缀，表示它们属于控制消息处理的一部分

  • CMSG_FIRSTHDR： 返回与给定 msghdr 结构体相关联的第一个控制消息头的指针。

  • CMSG_NXTHDR： 返回下一个控制消息头的指针，用于在消息头链表中遍历控制消息。

  • CMSG_LEN： 根据给定数据大小计算控制消息头的总长度。

  • CMSG_SPACE： 根据给定数据大小计算整个控制消息的总空间大小，包括头部和数据部分。

  • CMSG_ALIGN： 对给定大小进行对齐操作，确保满足平台的对齐要求。

  • CMSG_DATA: 是一个宏，用于获取控制消息头 (struct cmsghdr) 中数据部分的指针。在控制消息中，数据部分紧跟在头部之后，其大小可变。

msghdr 用于描述整个消息，包括数据和控制信息，而 cmsghdr 则用于描述控制信息的头部。在实际应用中，msg_control 指向一个或多个 cmsghdr 结构体，后面跟着实际的控制信息数据。这样的结构设计使得在消息中可以携带各种类型的辅助信息，例如文件描述符等。

sendmsg()

sendmsg() 函数用于通过套接字发送消息，包括数据块和控制信息。它允许在一个系统调用中发送多个数据块，同时可以携带控制信息，比如文件描述符。

  • sockfd：
    • 是要发送数据的套接字描述符。
  • msg：
    • 是一个指向 struct msghdr 结构体的指针，用于指定发送的消息的详细信息，包括目标地址、数据块、控制信息等。
  • flags：
    • 是一组标志位，可以为 0 或者通过按位或操作指定多个标志。
  • 返回值：
    • 返回实际发送的字节数，发生错误时返回 -1。

recvmsg()

recvmsg() 函数用于通过套接字接收消息，包括数据块和控制信息。它允许在一个系统调用中接收多个数据块，同时可以携带控制信息，比如文件描述符。

iovec

struct iovec 是一个用于描述分散/聚集输入输出的结构体，通常在一些系统调用中使用，如 readv、writev、sendmsg 和 recvmsg。
