键空间

在上文中提到了一个类型redisDb *,这是一个结构体，其中一个属性：

redis是一个键值对数据库，其中数据库由redisDb表示，其中的dict保存着数据库的所有键值对，这个dict称为：数据库键空间。键空间的键是字符串对象，键空间的值是任意一种Redis对象

增删改查的操作实际上是在操作键空间上的数据。其中增和改用同一个命令，比如字符串对象的SET，当命令后跟的键不在键空间时，则增加这个键，在键空间时则更改这个键对应的值。

过期键删除策略

redis采用定期删除和惰性删除结合的形式：

  • 定期删除：每隔一段时间删除过期键，并限制删除时长和频率减少删除操作对CPU的影响
  • 惰性删除：当访问到过期键时才删除

RDB

使用SAVE BGSAVE命令会创建一个.rdb文件，即RDB持久化功能。此时，过期的键不会被保存到.rdb文件中。

当载入rdb文件时，分主服务器和从服务器两种情况：

  1. 主服务器：.rdb文件中的过期的键不会载入
  2. 从服务器：.rdb文件中的所有键会被载入。当主服务器数据同步时，从服务器的数据库会被删除。

AOF

当键过期且被删除时，服务器向AOF文件发送DEL命令，记录该键已经删除。

AOF重写时，仅重写不过期的键

复制

客户端向主服务器读取一个键时，如果过期了，主服务器将删除这个键并返回空值。当主服务器删除了一个过期键时，此时主服务器也向其他的从服务器发送删除过期键的指令。

客户端向从服务器读取一个键时，即使读取了过期键也像正常的键一样处理，只有接收了主服务器发送的删除过期键指令才会删除。
