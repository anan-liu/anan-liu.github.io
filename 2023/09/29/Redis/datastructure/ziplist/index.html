压缩列表

列表键、哈希键中的值不大时使用压缩列表作为数据结构

实现

图源：www.redisbook.com

  • zlbytes:表示压缩列表所占字节数（十六进制表示）
  • zltail:一个偏移量，指向起始地址的指针加上这个偏移量就能指向最后一个节点
  • zllen:表示节点的数量
  • entry1 - entryn:节点
  • zlend:特殊标志，表明这是一个末尾

节点的结构

放大上一小节提到的entry，他详细的结构如下图：

图源：www.redis.com

  • previous_entry_length:上一个节点的长度。其中：如果上一个节点的长度 < 254 bytes，则该区域为1字节，这1字节可以表示上一节点的大小；如果上一个节点的长度 >= 254 bytes，则该区域为5字节，其中第一个字节为 0xFE,剩余4个字节表示上一节点的大小。

    假设有一个指针ptr指向entry4,它可以通过ptr - entry4.previous_entry_length,这样，ptr就指向了ptr3。从表尾遍历到表头就是通过这种机制遍历的。
