初始化配置

刚装上git的机器，需要做一个初始化。git的配置信息存储在三个文件中，三个文件的作用范围分别为：对所有用户生效，对当前用户生效，对当前仓库生效

  • 对所有用户生效：/etc/gitconfig,执行git config 带上 --system选项，命令的操作对象就是这个文件
  • 对当前用户生效：~/.gitconfig/,执行git config 带上 --global选项，命令的操作对象就是这个文件
  • 对当前仓库生效：.git/config,执行git config 带上 --local选项，命令的操作对象就是这个文件

仓库

仓库（Repository）是存储和管理项目代码的地方。它可以位于本地计算机上，也可以位于远程服务器上。

仓库状态

  • 已提交（committed）:已修改表示修改了文件，但还没保存到数据库中。

  • 已修改（modified） :已暂存表示对一个已修改文件的当前版本做了标记，使之包含在下次提交的快照中。

  • 已暂存（staged）:已提交表示数据已经安全地保存在本地数据库中。

以上是仓库的三种基本状态，但有个前提：仓库需要被跟踪才拥有以上的状态。因此有状态转移图如下：

图源：Git - 记录每次更新到仓库 (git-scm.com)

使用git status命令可以快速查看当前目录中的文件状态。

  1. 当新增了一个文件后，在没有使用git add之前，它是未被跟踪的状态：Untracked

    使用git add之后，它的状态转移为staged

    状态转移：Untracked->Staged

  2. 使用git commit将跟踪的文件提交到仓库，此时是:committed状态/unmodified状态

    状态转移：Staged->Unmodified

  3. 修改一个已跟踪的文件，文件的状态:Modified

    状态转移：Unmodified->Modified

  4. 一个Modified状态的文件，仍然需要使用git add将其转为：Staged，这样才能提交到仓库

    状态转移：Modified->Staged

使用git status打印出来的状态比较详细，繁琐。使用git status -short可以简短输出：

A代表这个文件在暂存区；M代表文件已修改但还未暂存；AM代表文件修改过暂存后又一次修改，但未暂存；??未跟踪。

忽略文件

在仓库主目录创建一个.gitignore文件，里面存放着被忽视的文件。可以使用glob模式匹配。

命令

命令 选项 作用
git add 将一个文件添加到下一次提交
git commit -a/-m 提交文件，会启动一个编辑器用以描述提交说明
git status -s 显示当前目录下的各个文件状态
git diff –staged 比较暂存区域和目录下文件的差别
git rm –cached 删除暂存区域中的文件且从磁盘中删除它
git mv 和mv命令作用一样，但这里它同样作用于暂存区域的文件
