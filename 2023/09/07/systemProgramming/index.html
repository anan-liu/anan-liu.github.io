· errno

系统库函数中的一个全局变量，记录最近一个错误的错误号。

· perror

在标准库函数中，描述errno中的错误号并打印这段描述。函数原型为：

void perror(const char* s);

s是程序员传入的字符串，最终的打印结果为：字符串:错误描述

· umask

一个权限掩码，用于限制新创建文件的权限。只影响新创建的文件而不影响已有文件的权限。

已有文件的权限可以通过chmod命令来做调整

系统I/O相关函数

· open()

使用open函数，首先需要包含以下3个头文件

open的原型如下：

c 语言是没有函数重载的，这里是运用了可变参数

open函数如果成功打开了文件，则返回一个文件描述符。失败则返回-1.

· open()-1

首先解析第一个：int open(const char *pathname, int flags);

pathname：要打开文件的路径

flags：对文件的操作设置和其他设置。查阅帮助文档man open，其中规定了必选项：

  • O_RDONLY
  • O_WRONLY
  • O_RDWR

他们的作用顾名思义，分别为只读，只写，读写

此外，帮助文档给出了可选项，典型：

  • O_CREAT：没有这个文件则新建文件
  • O_APPEND：写文件时是在文件后追加内容而不是覆盖他

· open()-2

现在解析第二个：int open(const char *pathname, int flags, mode_t mode);

前两个参数作用是一样的，多了个mode的作用：

  • mode：传入8进制的数，用以限制新建文件的权限

    注意，mode的8进制数并非最终文件的权限，它还需要和umask的反码做与运算。

如果open函数没有创建任何文件，则mode参数被忽略

· flags参数

除了传入必选项，同时还需要传入了可选项，它的形式应该如下：

int fd = open("create.txt", O_RDWR | O_CREAT, 0777);

flags参数时int类型，有32位。其中32位的每一位对应了一个功能，必选项和可选项的功能都有具体的代表位，当那个位置1时则生效。因此当需要多个可选项时，我们对其做或运算，这样他们的代表位都置1了，功能也就生效了

· read()

函数原型为：ssize_t read(int fd,void *buf,size_t count);

参数的用处：

  • fd：open函数打开指定文件所返回的文件描述符
  • buf：缓冲区，读取的数据存放地。是一个指针，通常指向数组起始地。
  • count：数组的大小，当读取的数据最大不大于这个值

read函数读取成功时返回读到的字节数，失败则返回-1

· write()

函数原型为：ssize_t write(int fd, const void *buf, size_t count);

参数的用处和read()类似，不过读取变成了写入

read 和 write都包含在#include <unistd.h>中

· lseek()

包含在#include <sys/types.h> #include <unistd.h>中，其中第一个include是定义了宏，第二个包含了lseek()

函数原型为：off_t lseek(int fd, off_t offset, int whence);

参数：

  • offset：指针偏移量
  • whence：决定指针从何处偏移：
    • SEEK_SET：从文件起始处偏移
    • SEEK_CUR：从当前指针偏移
    • SEEK_END：从文件末端偏移

返回值：函数执行成功返回偏移量，出错返回-1

注意：

  • 如果whence参数采用了SEEK_END,也就是从文件末端偏移，且偏移大于2，此时指针和文件末端最后一个字符之间会有一些空值，这些空值不占据磁盘空间。
  • 测试lseek()函数是否执行正确应当测试它是否等于-1，因为一些设别运行其偏移量设置为负数，然后函数执行完毕后会成为返回值

RIO包(CSAPP)

read()、write()函数有时候会传送比要求的少的字节。称为不足值，在网络应用程序中，需要不断调用读或者写函数处理不足值，直到把规定的字节传送完毕。这时候可以使用RIO包函数，它可以自动处理不足值。

他有两类函数：

  • 带缓冲的输入函数
  • 不带缓冲的输入输出函数

· stat()&lstat()

stat结构体：

原型：

stat命令的功能是显示一个文件的详细信息。

stat()函数的作用是将一个文件的详细信息传出给buf参数，pathname参数指定一个文件的路径。

stat() 和 lstat()基本相同，不同的点：当stat()遇到了一个软链接，他会显示软链接指向的文件的详细信息，而lstat()则显示软链接的信息。

restrict:只能用在函数的指针类型的形参上，当指针类型的形参遇上这个关键字restrict时，编译器认为形参中的所有指针指向的地方都不相同，这样编译器可以更偏激的优化代码。参考关键字restrict
